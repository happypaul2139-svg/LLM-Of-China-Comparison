# 场景二：写代码

## 任务

请开发一个「家庭旅行费用 AA 记账本」网页应用。场景：一家三口（爸爸、妈妈、爷爷）和两个朋友家庭一起出去旅行，旅途中各种消费由不同人垫付，最后需要算清楚谁欠谁多少钱。

## 技术要求

- 纯前端实现，单个 HTML 文件，可直接用浏览器打开运行
- 可以使用 CDN 引入的第三方库（如 Vue、React、Chart.js 等），但不依赖后端服务
- 数据存储在 localStorage 中，刷新页面不丢失
- 页面需适配手机端浏览器

## 功能要求

### 1. 成员管理

- 支持添加和删除旅行成员，每个成员有姓名和所属家庭（如"张家""李家""王家"）
- 预置三个家庭共 7 人的示例数据

### 2. 记账功能

- 支持添加一笔消费记录，包含：日期、消费描述、金额、垫付人（单选）、参与人（多选，默认全选）
- 消费记录以时间倒序列表展示，支持删除
- 每条记录显示人均分摊金额

### 3. 结算面板

- 实时显示每个人的总垫付金额和总应付金额
- 计算最终结算方案：用最少的转账笔数算出"谁需要转给谁多少钱"
- 结算结果以清晰的列表展示，如："李妈妈 → 张爸爸 320 元"

### 4. 统计视图

- 按家庭汇总总消费金额，用柱状图或饼图展示各家庭消费占比
- 按消费类别统计（如果记录中有类别的话），展示消费结构
- 显示本次旅行的总花费和人均花费

### 5. 数据导出

- 支持将所有消费明细导出为 CSV 文件并下载
- CSV 包含：日期、描述、金额、垫付人、参与人、人均金额

## 验收标准

1. HTML 文件可直接在 Chrome 浏览器中打开，无报错，页面正常渲染
2. 能正常添加和删除成员及消费记录
3. 结算算法正确：所有人的收支相抵后净额之和为零
4. 最少转账笔数的结算方案合理
5. 统计图表数据与实际记录一致
6. CSV 导出功能正常，文件内容完整
7. 刷新页面后数据不丢失
8. 手机端浏览器上布局不错乱，可正常操作
